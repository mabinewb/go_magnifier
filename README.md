# Go Magnifier

Go 언어로 작성된 Windows 전용 오버레이 도구입니다. 화면 캡처, 이미지, 텍스트를 각각 별도 오버레이로 띄울 수 있고, 오버레이마다 위치, 크기, 투명도, 클릭 통과 여부를 독립적으로 조절할 수 있습니다.

현재 앱은 단일 확대기보다는 여러 개의 오버레이를 동시에 관리하는 데 초점이 맞춰져 있습니다. 화면 일부를 확대해서 띄우는 용도뿐 아니라, 고정 이미지 패널이나 텍스트 패널을 항상 위에 두는 용도로도 사용할 수 있습니다.

## 지원 기능

- 여러 개의 오버레이 동시 생성 및 개별 편집
- 화면 캡처, 이미지 파일, 텍스트 소스 전환
- 오버레이 드래그 이동 및 가장자리 리사이즈
- 투명도 조절
- 마우스 클릭 통과
- 화면 캡처 리프레시율 조절
- 캡처 백엔드 선택
- 자동, GDI BitBlt, Desktop Duplication (DXGI)
- Desktop Duplication 사용 시 재귀 캡처 차단
- 텍스트 오버레이 편집
- 글꼴, 글자 크기, 굵게, 기울임, 정렬, 색상, 배경색, 배경 투명도, 테두리 설정
- 설정 자동 저장 및 다음 실행 시 복원
- 설정 파일 내보내기 및 불러오기
- 설정 폴더에 실행 로그 자동 저장

## 실행 환경

- Windows 전용
- 멀티 모니터 환경 지원
- 음수 좌표 모니터 배치 지원

## 빠른 실행

PowerShell에서 프로젝트 루트로 이동한 뒤 실행합니다.

```powershell
.\scripts\run.ps1
```

이 스크립트는 다음을 처리합니다.

- Go 사용 가능 여부 확인
- 필요 시 Go 다운로드
- 의존성 준비
- 앱 빌드
- dist/GoMagnifier.exe 실행

## 빌드

실행 파일만 만들고 싶다면 아래 스크립트를 사용합니다.

```powershell
.\scripts\build.ps1
```

빌드 결과물은 dist/GoMagnifier.exe 에 생성됩니다.

자주 쓰는 예시는 아래와 같습니다.

```powershell
.\scripts\build.ps1 -Clean
.\scripts\run.ps1 -Rebuild
.\scripts\run.ps1 -ForceGoDownload
```

## 사용 방법

1. 앱을 실행합니다.
2. 오버레이를 하나 이상 추가합니다.
3. 소스 종류를 화면 캡처, 이미지 파일, 텍스트 중 하나로 선택합니다.
4. 화면 캡처 소스라면 캡처 영역 선택으로 원본 영역을 지정합니다.
5. 선택기에서 새 영역을 드래그해 만들 수 있고, 기존 파란 영역의 가장자리를 다시 드래그해 크기를 조절하거나 내부를 드래그해 위치를 옮길 수 있습니다.
6. 오버레이 창 자체는 화면에서 직접 이동하거나 크기를 바꿀 수 있습니다.
7. 필요에 따라 투명도, 확대 비율, 클릭 통과, 리프레시율, 백엔드, 텍스트 속성을 조절합니다.

## 캡처 백엔드

화면 캡처 소스는 세 가지 모드를 지원합니다.

- 자동
	Desktop Duplication (DXGI)을 먼저 시도하고 실패하면 GDI BitBlt로 대체합니다.
- 기존 GDI BitBlt
	호환성이 더 넓지만 성능과 지연 측면에서는 불리할 수 있습니다.
- Desktop Duplication (DXGI)
	일반적으로 더 효율적이며, 재귀 캡처 차단 기능은 실제로 이 백엔드가 사용 중일 때만 동작합니다.

초기 프레임 준비 시점에는 Desktop Duplication이 일시적으로 프레임을 아직 주지 못할 수 있습니다. 이 경우 즉시 화면 이상으로 이어지지 않는 짧은 과도 상태일 수 있습니다.

## 설정 및 로그

앱은 사용자 설정 폴더 아래에 설정과 로그를 자동 저장합니다.

- settings.json
	현재 오버레이 상태와 앱 설정
- runtime.log
	실행 중 상태 메시지와 오류 로그

앱의 설정 폴더 열기 버튼으로 바로 접근할 수 있습니다.

## 트러블슈팅

- 화면 캡처 결과가 기대와 다르면 먼저 캡처 백엔드를 자동과 GDI, DXGI 사이에서 바꿔 봅니다.
- 재귀 캡처 차단은 실제 백엔드가 DXGI일 때만 동작합니다.
- 문제 제보 시 settings.json 과 runtime.log 를 함께 확인하면 원인 파악이 빠릅니다.
- 캡처 영역을 매우 작게 잡을 수도 있지만, 지나치게 작은 영역은 사용 목적에 따라 가독성이 떨어질 수 있습니다.

## 프로젝트 구조

- cmd/magnifier
	프로그램 진입점
- internal/ui
	설정창, 오버레이 창, 선택기, 런타임 로그 처리
- internal/capture
	화면, 이미지, 텍스트 캡처 및 렌더링
- internal/model
	설정 모델과 정규화 로직
- internal/persist
	설정 파일 저장 및 복원
- scripts
	Go 부트스트랩, 실행, 빌드 자동화

## 제한 사항

- Windows 전용입니다.
- 일부 시스템이나 드라이버 환경에서는 DXGI Desktop Duplication 동작이 달라질 수 있습니다.
- 게임 캡처나 보호된 콘텐츠 캡처는 일반 데스크톱 캡처와 동일하게 동작하지 않을 수 있습니다.
